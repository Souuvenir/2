Explicacion Codigo:
Para comenzar la codificación de esta evaluación como siempre 
comencé leyendo el enunciado cuidadosamente e identificando 
las, clases, variables, métodos a utilizar y se asignan las herencias.
Comenzando por la codificación de la clase Microsoft:
logre identificar que todas las variables que tuvieran solo 
2 estados, en este caso encendido y apagado serian booleans.
validándose los estados correspondientes con los requerimientos
con un if, los métodos a utilizar los declaré como String para así
poder retornar los estados en cadenas de texto en pantalla
sin la necesidad de usar system.out.println.
Al método onHuella decidí agregarle una variable por parámetro
para poder validar apropiadamente si el lector está en uso. Dedo en 
el lector true simula de mejor forma el flujo de acciones 
de un lector real.
Los requerimientos de digitalpersona se codifican de manera similar
con booleans y el tipo string en los métodos se mantienen para poder
visualizar mejor la información de los estados en pantalla.
Con respecto a Apptest
Se crean los 2 objetos con variables de referencia de tipo biométricos 
llamados b1 y b2 
para efectos de poder mostrar los retornos de los métodos string se crean 
unas variables llamadas resultado a las cuales les asigno el valor de retorno
de cada uno de los métodos de las clases a utilizar.
posteriormente a eso se llaman los métodos y se imprimen en pantalla 
con su respectivo nombre, clase a la que pertenece y el resultado.
